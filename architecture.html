Command Flow

User sends message in Telegram

"switch to channel 105"

Telegram forwards to backend via webhook

POST /webhook/telegram — JSON update payload

Access control check

User ID verified against ALLOWED_TELEGRAM_IDS whitelist

LLM Agent parses natural language → JSON

Claude Haiku returns {"command":"change_channel","value":105}

Backend broadcasts JSON over WebSocket

Sent to all authenticated TV clients (or a specific device_id)

Android TV client receives command

Kotlin service running as persistent foreground process

KeyEvent injection via Accessibility Service

KEYCODE_1 → KEYCODE_0 → KEYCODE_5 (200ms apart)

Telegram bot sends confirmation to user

"📺 Switching to channel 105"

Key Code Samples

1 · LLM System Prompt (structured extraction)

The prompt constrains the model to return only a JSON object:

system_prompt.txt
# Sent to Claude Haiku as the system prompt
You are a TV remote control assistant.
Parse the user message and return ONLY valid JSON — no markdown.

Supported shapes:
  change_channel  → {"command": "change_channel", "value": <int>}
  channel_up      → {"command": "channel_up"}
  volume_up       → {"command": "volume_up"}
  mute            → {"command": "mute"}
  power           → {"command": "power"}
  unknown         → {"command": "unknown", "message": "..."}

2 · WebSocket auth handshake

The TV client sends a shared secret immediately after connecting:

sequence
# Android TV → Server
{"auth": "change-me-use-a-long-random-string"}

# Server → Android TV (success)
{"status": "authenticated"}

# Server → Android TV (failure → connection closed 1008)
{"error": "unauthorized"}

4 · Kotlin KeyEvent injection (core snippet)

TvRemoteAgent.kt
fun injectKey(keyCode: Int) {
    val down = KeyEvent(KeyEvent.ACTION_DOWN, keyCode)
    val up   = KeyEvent(KeyEvent.ACTION_UP,   keyCode)
    val im   = InputManager.getInstance()
    im.injectInputEvent(down, 0)
    im.injectInputEvent(up,   0)
}

// Channel 105 → three sequential key presses
"105".forEach { digit ->
    injectKey(digitToKeyCode(digit))
    Thread.sleep(200)
}

Security

🔐

Telegram User Whitelist

Set ALLOWED_TELEGRAM_IDS to a comma-separated list of numeric Telegram user IDs. All other users receive an "unauthorized" reply.

🔑

WebSocket Shared Secret

The TV client must present TV_CLIENT_SECRET in the first WebSocket message. A wrong or missing secret closes the connection immediately.

🔒

TLS Everywhere

Deploy behind NGINX or a reverse proxy (Caddy is easiest) with a valid TLS certificate. Use wss:// WebSocket URLs, never plain ws://.

🌐

Webhook Secret Token

Configure Telegram's secret_token when calling setWebhook and verify the X-Telegram-Bot-Api-Secret-Token header in every request.

♻️

Auto-Reconnect

The Android TV client retries the WebSocket connection after 5 seconds on any failure and restarts automatically on device reboot via BOOT_COMPLETED broadcast.

📋

Rate Limiting (optional)

Add per-user rate limiting in the Telegram webhook handler (e.g. max 10 commands/minute) to prevent spam or accidental loops.

Deployment Guide

  • Server: Deploy on any $5/month VPS (Fly.io, Railway, Render, DigitalOcean) — the backend is stateless except for active WebSocket connections. Use the provided Dockerfile.
  • HTTPS/WSS: Use Caddy (caddy reverse-proxy --from yourdomain.com --to localhost:8000) for automatic TLS — it handles Let's Encrypt for you.
  • Telegram Webhook: Run python register_webhook.py once after deployment to point Telegram at your server.
  • Android TV App: Sideload the APK via ADB or a file manager. Enable the Accessibility Service once in Settings → Accessibility.
  • Auto-start: The BootReceiver starts the foreground service automatically after every reboot — no manual action needed.
  • Environment variables: Copy .env.example → .env, fill in the four values, and pass them to Docker via --env-file .env.
  • Multiple TVs: Each Android TV connects with a unique device_id in the WebSocket URL. The /devices bot command lists all currently connected TVs.
